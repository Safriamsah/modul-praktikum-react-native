# Tutorial 2: Understanding Project Structure

## Overview

In this tutorial, we'll explore the files and folders in your Expo project and understand their purposes.

## Project Files Breakdown

### ğŸ“„ App.js
The main application component - your app starts here. This is where you'll write most of your application logic.

```javascript
export default function App() {
  // Your app code here
}
```

**Key Points:**
- This is the root component
- Everything you want to display must be returned from this component
- You can split code into multiple files and import them here

### ğŸ“„ package.json
Defines your project dependencies and scripts.

```json
{
  "name": "expoJS-template",
  "scripts": {
    "start": "expo start",
    "android": "expo start --android",
    "ios": "expo start --ios",
    "web": "expo start --web"
  },
  "dependencies": {
    "expo": "~54.0.27",
    "react": "19.1.0",
    "react-native": "0.81.5"
  }
}
```

**Important Scripts:**
- `npm start` - Start development server
- `npm run android` - Run on Android
- `npm run ios` - Run on iOS
- `npm run web` - Run in web browser

### ğŸ“„ app.json
Expo configuration file for your app settings.

```json
{
  "expo": {
    "name": "expoJS-template",
    "slug": "expojs-template",
    "version": "1.0.0",
    "orientation": "portrait"
  }
}
```

**Configurable Options:**
- App name and version
- Icons and splash screens
- Platform-specific settings
- Permissions and features

### ğŸ“„ index.js
Entry point that registers your main App component.

```javascript
import { registerRootComponent } from 'expo';
import App from './App';

registerRootComponent(App);
```

You typically don't need to modify this file.

### ğŸ“‚ node_modules/
Contains all installed npm packages. Never edit these files directly!

This folder is:
- Generated by `npm install`
- Ignored by git (listed in .gitignore)
- Can be deleted and regenerated anytime

### ğŸ“„ .gitignore
Specifies which files should not be tracked by git.

Common ignored files:
- `node_modules/`
- `.expo/`
- `*.log`
- Build artifacts

## Creating a Component Folder

Let's create a simple folder structure:

```bash
mkdir components

```

### Example: Creating a Component

Create a file `components/Header.js`:

```javascript
import { View, Text, StyleSheet } from 'react-native';

export default function Header({ title }) {
  return (
    <View style={styles.header}>
      <Text style={styles.title}>{title}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  header: {
    padding: 20,
    backgroundColor: '#007AFF',
  },
  title: {
    fontSize: 24,
    color: 'white',
    fontWeight: 'bold',
  },
});
```

### Using the Component in App.js:

```javascript
import Header from './components/Header';

export default function App() {
  return (
    <View style={styles.container}>
      <Header title="My App" />
      <Text>Welcome!</Text>
    </View>
  );
}
```


**Note:** You'll need to install `react-native-dotenv` to use .env files.

## Best Practices

1. **Keep App.js Clean**: Use it as a container for routing and main structure
2. **Separate Concerns**: Create folders for different types of files
3. **Naming Conventions**: 
   - Components: PascalCase (Button.js)
   - Utilities: camelCase (apiHelper.js)
   - Constants: UPPER_CASE
4. **One Component Per File**: Makes code easier to find and maintain
5. **Group Related Files**: Keep component styles close to components
